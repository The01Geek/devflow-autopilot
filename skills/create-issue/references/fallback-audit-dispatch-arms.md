**Embed arm (the on-disk draft path is untrusted here).** When `query-arm` answers `arm=embed`, the dispatch prompt instead **embeds the full rendered draft title and body verbatim — carried in full, never condensed or summarized** — as the sole draft source, and it does **not** carry the read-the-file instruction. On this arm the out-of-bounds declaration names exactly these 6 files — `.devflow/tmp/issue-derivation-<slug>.md`, `.devflow/tmp/issue-draft-<slug>.md`, `.devflow/tmp/issue-audit-<slug>.md`, `.devflow/tmp/issue-audit-state-<slug>.json`, the **retired** `.devflow/tmp/issue-audit-state-<slug>.md`, and any staged canonical-draft artifact `.devflow/tmp/issue-draft-<slug>.*.staged.md`: the on-disk `issue-draft-<slug>.md` is untrusted on **every** entry path this arm has (it can only be a stale prior-run leftover or absent entirely; or the auditor's read of it already failed; or the file arm's identity check is unavailable), so this arm ignores the on-disk file and audits the verbatim embedded body instead — treating it as out of bounds is the safe, conservative choice on all of them. The two state paths are named for the reason the file arm names them: either one re-anchors the auditor on this run's prior verdicts and revision records.

This arm carries its own **carriage check** an LLM can discharge exactly: **the tool generates the two sentinel tokens** and prints them from `record-dispatch` (`sentinel_open=` / `sentinel_close=`). Bracket the embedded body with **exactly those printed tokens** — never tokens you choose yourself, which would compare against a value the tool never recorded — and require the auditor to **quote both sentinels plus the body's first and last lines verbatim** in its return. **Forward the quoted pair to `record-return --carriage-sentinel-open <quoted> --carriage-sentinel-close <quoted>` and obey the classification returned**; the tool owns the comparison and fails closed on an absent tuple exactly as the file arm does. Omit the flags when the return quoted no sentinels. The Step 4 audit summary line carries the corresponding embed marker, which the tool records from the `--marker` you passed at dispatch and re-emits through `query-summary` — the three markers stay distinct so the breadcrumb never falsely claims a failed write on a path where the write actually landed.

**Retry escalations (each stays inside the round it retries).**

- **DRAFT-UNREADABLE retry:** `record-return --verdict DRAFT-UNREADABLE` → `query-next-action` → `dispatch-embed-retry` → `record-dispatch --arm embed --marker file-unreadable --round "<round>"` (bytes on stdin, **same round** — no second round record); the embed arm hard-requires `--marker`, and `file-unreadable` is the marker `query-arm` answers for a prior `DRAFT-UNREADABLE`.
- **File-arm same-arm retry escalating to embed:** on a `dispatch-retry-same-arm` answer for a **file-arm** round, the canonical file can become unhashable between the return and the retry (a concurrent overwrite/delete). The tool permits escalating that retry to the **embed** arm rather than deadlocking — and the embed arm hard-requires `--marker`, so re-run `query-arm` and pass **the marker it answers**: `digest-unrecorded` when the file is now unhashable, or `file-unreadable` when a prior `DRAFT-UNREADABLE` is recorded (bytes on stdin, **same round**). The escalation is never silent — the marker is recorded in `embed_markers` and rendered in the summary, exactly like the `dispatch-embed-retry` escalation above.

**Degraded arm — attempt-first, never pre-detected.** Dispatch **whenever a subagent tool is exposed** (nested contexts included; nested dispatch was live-probe-verified working at drafting time — never pre-detect a nested context and skip). You reach this arm when the runner exposes **no** subagent-dispatch tool, when the dispatch **call itself errors**, or when `query-next-action` answers `dispatch-inline-degraded` — a *still-running* dispatch is awaited, never retried as a bad return. On the degraded arm, run the **same audit prompt inline** against the rendered draft, then record it (`record-dispatch --arm inline --round <round>` with the bytes on stdin, plus `record-degraded --round <round> --reason …` — each carries `--round` per the requirement the Degraded/inline bullet states) so the summary line is marked degraded — **a skipped audit is never silent.** The degraded inline arm audits the **rendered draft already in context** and is **exempt from the read-the-file instruction** (there is no subagent to read it) and emits **no `VERDICT: DRAFT-UNREADABLE`** — that third value belongs to the file arm only, and the "degraded" summary marker already signals this arm's weaker properties.
