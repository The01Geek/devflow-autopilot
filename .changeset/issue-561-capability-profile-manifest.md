The review profile is locked as a security boundary by `lib/review-profile.tokens`, so widening the read-only reviewer always requires a deliberate, visible diff, and `--check` refuses a region with a duplicated anchor (an injected second assignment that would win at runtime) rather than verifying only the canonical copy. This wave also eliminates the live runnerâ†”probe review-pair drift (the probe `REVIEW` baseline was missing two vendored-helper grants).
