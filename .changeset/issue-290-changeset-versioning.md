- **DevFlow now versions itself with changesets instead of an in-PR `plugin.json` + `CHANGELOG.md` bump, ending the version/CHANGELOG merge conflict that taxed every concurrent PR.** Each engine-surface PR adds a uniquely-named `.changeset/*.md` file (a `bump: patch|minor|major` frontmatter key plus Keep-a-Changelog prose) and no longer edits `.claude-plugin/plugin.json` or `CHANGELOG.md`. On merge to `main`, the new `version-consolidate` GitHub Action (`.github/workflows/version-consolidate.yml`) runs `scripts/consolidate-changesets.py`, which consumes every pending changeset: it bumps `plugin.json`'s `version` by the highest pending bump type (patch < minor < major — one increment even when several are pending), prepends a single dated, PR-cited entry assembled from all the pending prose, deletes the consumed changeset files, and commits with the load-bearing `chore: bump version` subject that `docs-release-notes` Step 4b still keys on. A malformed changeset (missing/invalid `bump`, unparseable frontmatter, empty prose) fails the Action loudly and names the offending file — never a silent skip or a partial write; with no pending changesets it is a clean no-op. The `/devflow:implement` versioning policy, the Phase 3 review gate (now "engine-surface change ⇒ changeset present"), the docs, and the `lib/test/run.sh` coupling pins all move together. (#293)
